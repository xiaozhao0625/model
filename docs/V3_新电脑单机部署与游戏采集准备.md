# V3 新电脑单机部署与游戏采集准备

## 1. 硬件要求

- Windows PC。
- 建议 NVIDIA 独立显卡，当前已验证 RTX 4060 Laptop GPU 可用。
- 建议 16GB 以上内存，模型与 OCR 缓存建议预留 50GB 以上磁盘空间。

## 2. Windows 要求

- 目录统一使用 `D:\work\app-shot`。
- 采集期间需要保持当前桌面可交互。
- 采集期间可临时不息屏/不睡眠，采集结束后恢复原电源策略。

## 3. GPU 要求

- 安装 NVIDIA 显卡驱动。
- `nvidia-smi` 应可用。
- PaddleOCR GPU 环境优先使用 `D:\work\app-shot\venvs\v3-gpu`。

## 4. 目录结构

```text
D:\work\app-shot
  cache
  downloads
  logs
  model
  models
  obs-output
  reports
  runs\v3
  tools
  venvs
```

## 5. 需要下载的软件

- Git。
- Python 3.11 或当前项目验证版本。
- Node.js LTS + npm。
- OBS Studio，放到 `D:\work\app-shot\tools\obs-studio`。
- FFmpeg，放到 `D:\work\app-shot\tools\ffmpeg`。
- ShowUI 模型权重，放到 `D:\work\app-shot\models\showui`。
- 目标软件或单机游戏，放到 `D:\work\app-shot\tools` 或 `D:\work` 下。

## 6. 不需要的组件

- Redis：当前 V3 单机模式不需要安装。
- PostgreSQL：当前 V3 单机模式不需要安装。
- Docker：当前 V3 单机模式不需要安装。
- 旧 Master/Worker 分布式服务：当前 V3 单机采集不需要。

旧 `deploy` 目录里若出现 Redis/PostgreSQL/Docker 字样，只代表历史或分布式部署材料，不允许阻塞 V3 单机启动。

## 7. 部署脚本用法

```powershell
cd D:\work\app-shot\model
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\v3\deploy\setup_v3_app_shot_node.ps1 `
  -Root D:\work\app-shot `
  -RepoUrl https://github.com/xiaozhao0625/model.git `
  -Branch feat/v3-obs-ocr-model-agent `
  -InstallPythonDeps `
  -InstallNodeDeps
```

如仓库已经准备好，可以加 `-SkipGitPull`。

部署报告：

```text
D:\work\app-shot\reports\v3_deploy_report.json
D:\work\app-shot\reports\v3_deploy_report.md
```

## 8. 模型权重放置方式

- PaddleOCR 模型缓存：`D:\work\app-shot\models\paddleocr`。
- ShowUI 权重：`D:\work\app-shot\models\showui`。
- Hugging Face / torch / paddle 缓存：`D:\work\app-shot\cache` 下对应目录。

## 9. 启动方式

后端：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\v3\start\start_v3_backend_app_shot.ps1
```

前端：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\v3\start\start_v3_web_app_shot.ps1
```

全部启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\v3\start\start_v3_all_app_shot.ps1
```

前端端口固定为 `5173`，访问地址：

```text
http://localhost:5173/v3
```

如果 5173 被占用，脚本会报错并停止，不会悄悄换端口。

## 11. 游戏采集安全边界

默认只允许低风险模式：

- 单机游戏。
- 离线游戏。
- 训练场。
- 菜单界面。
- 自己本地安装的游戏。

禁止：

- 自动登录。
- 自动验证码。
- 自动支付/充值。
- 自动购买。
- 自动聊天。
- 自动匹配真人。
- 自动排位。
- 绕过反作弊。
- 刷资源/刷金币。
- 修改游戏文件或注入进程。

默认配置：

```text
game_mode_ready=false by default
enable_game_explorer=false by default
```

只有用户明确开启后，才允许键鼠探索。

## 12. 常见问题

- `/v3` 页面不可访问：确认后端 `8000` 和前端 `5173` 已启动。
- 前端端口被占用：关闭占用 5173 的进程，不要换端口。
- OCR 慢：运行 `scripts\v3\model\check_ocr_gpu_performance_app_shot.ps1`，确认 GPU OCR ready。
- ShowUI 不可用：检查 `D:\work\app-shot\models\showui` 是否存在权重。
- 输入网关不可用：确认屏幕未锁定、未息屏、当前桌面可交互。

## 13. 电源策略

真实采集期间可以临时关闭息屏/睡眠。任务结束后应恢复原电源策略，除非用户明确要求继续保持不息屏。
