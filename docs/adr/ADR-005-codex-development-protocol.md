# ADR-005：Codex 开发协议

## 状态

Accepted

## 背景

项目涉及多类型自动化、模型决策、截图采集、上传确认和本地删除。需要明确 Codex 的执行边界，避免越权实现和不可审计变更。

## 决策

- 每次先给计划，再改文件。
- 小步实现，只改相关文件。
- 核心逻辑必须有测试。
- 不伪造测试结果。
- 完成后必须给出命令和输出。
- 遇到架构不清或测试失败，先停下说明。
- P0 只建立项目文档、阶段计划、Codex 工作协议和架构基线。

## 影响

- 所有阶段都必须先对齐范围和验收。
- Codex 不得擅自新增依赖、改 UI、接模型或改业务架构。
- 代码阶段必须执行最小验证命令并报告真实结果。
