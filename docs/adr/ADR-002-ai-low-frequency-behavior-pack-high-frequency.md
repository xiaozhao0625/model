# ADR-002：AI 低频决策，行为包高频执行

## 状态

Accepted

## 背景

截图采集需要稳定、可审计、可重复。让 AI 做高频逐帧操作会带来成本、延迟、不可控和审计困难。

## 决策

- AI 只做启动、场景判断、按钮定位、卡住恢复等低频决策。
- 高频动作由自动化工具、行为包、ADB、Playwright、pywinauto、OBS/FFmpeg 等机制执行。
- PC 游戏 high 桶必须使用行为包 + OBS/FFmpeg 抽帧。
- 不依赖闭源 Computer Use API 作为核心能力。

## 影响

- P3 模型网关只能承担低频决策。
- P4 Worker 和行为包必须承担高频采集。
- 行为包运行必须可审计、可复现、可回滚。
