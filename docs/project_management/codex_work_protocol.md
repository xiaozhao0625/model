# Codex 工作协议

## 基本原则

Codex 是本项目的执行工程师，必须严格按架构师通过用户转发的任务执行。Codex 不擅自改架构、不扩范围、不引入无关依赖。

## 每次任务流程

1. 先查看当前仓库结构和相关文件。
2. 先给计划，再改文件。
3. 小步实现，只改相关文件。
4. 不修改任务范围外的业务代码。
5. 不新增依赖，除非任务明确要求且架构师确认。
6. 核心逻辑必须有测试。
7. 不伪造测试结果。
8. 完成后必须给出执行命令和输出摘要。
9. 遇到架构不清、测试失败、约束冲突时，先停下说明。

## 禁止行为

- 禁止把闭源 Computer Use API 作为核心能力。
- 禁止让 AI 做高频逐帧操作。
- 禁止自动处理验证码、支付、充值、购买、聊天发送、账号安全验证、反作弊绕过。
- 禁止在用户确认上传百度网盘前删除本地图片和临时视频。
- 禁止删除 summary.json、meta.jsonl、upload_manifest.json、run.log。
- 禁止在 P0 写业务代码、接模型、改 UI、新增依赖。

## 验证要求

- 文档阶段至少运行文件清单和仓库状态命令。
- 代码阶段必须运行对应最小测试。
- 如果仓库存在 pytest、package.json、前端脚本或其他测试入口，必须选择与本次改动相关的最小验证命令。
- 命令失败时必须报告失败输出，不得改写为成功。
