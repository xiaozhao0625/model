# P5：补采机制与人工补种子

## 目标

建立补采、失败恢复、人工补种子和任务再运行机制，保证有效图片数量和场景覆盖质量。

## 范围

- valid_total 不足时的补采计划。
- 重复率或质量不足时的补采计划。
- 人工补种子入口。
- 补采审计。
- 补采后的状态回写。

## 核心规则

- valid_total < 1000 不得进入 capture_completed。
- valid_total > 5000 不得继续采集。
- 补采不得触发禁止自动处理场景。
- 人工补种子必须留痕。

## 不做

- 不自动绕过登录安全验证。
- 不自动发送聊天内容。
- 不自动购买、充值或支付。

## 验收标准

- 能识别不足、重复、低质量数据。
- 能生成补采任务。
- 人工种子来源可审计。
