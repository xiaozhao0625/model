# P3：模型网关

## 目标

建立低频 AI 决策网关，支持启动、场景判断、按钮定位、卡住恢复等决策，同时保持高频操作由自动化和行为包执行。

## 范围

- 模型网关接口。
- 模型调用审计。
- 决策类型枚举。
- 超时、失败、降级策略。
- 开源工具和开源模型优先策略。

## 核心规则

- AI 只做低频决策。
- AI 不做高频逐帧操作。
- 模型网关不得绕过禁止操作边界。
- 不依赖闭源 Computer Use API 作为核心能力。

## 不做

- 不训练模型。
- 不绑定单一闭源模型。
- 不接 PC 游戏 high 桶高频控制。

## 验收标准

- 决策调用可审计。
- 高频循环不依赖模型调用。
- 禁止场景能够停止并交给人工。
