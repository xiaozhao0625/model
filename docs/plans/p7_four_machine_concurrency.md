# P7：四机并发与生产化压测

## 目标

支持四机并发采集、任务调度、资源隔离、生产化压测和运行质量监控。

## 范围

- 四机任务分发。
- 节点心跳。
- 资源占用记录。
- 采集吞吐统计。
- 失败率、重复率、质量分布统计。
- 压测报告。

## 核心规则

- 每台机器必须遵守统一状态机。
- 本地暂存和清理规则不因并发而放宽。
- 上传确认仍由用户确认后进入 uploaded_confirmed。
- 删除后仍必须保留 summary.json、meta.jsonl、upload_manifest.json、run.log。

## 不做

- 不引入未确认的中心化闭源控制能力。
- 不牺牲审计日志换取吞吐。
- 不绕过禁止自动处理场景。

## 验收标准

- 四机并发任务状态一致。
- 异常节点不破坏其他节点任务。
- 压测报告覆盖吞吐、失败率、质量和数据完整性。
